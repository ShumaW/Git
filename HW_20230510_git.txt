1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?
  Глобальные настройки для github (делается один раз на новом устройстве)
  git config --global user.name 'name'
  git config --global user.email 'email'
  git config --global core.editor 'code --wait'
  создать новый каталог
  cd <имя нового каталога> - переход в созданный каталог  
  git remote add origin <скопированная_ссылка> (делает привязку локального репо к репо на GitHub)
  git clone - клонирование репозитория с github

2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

  1. Создать новый каталог (подкаталог)
  2. Создать в данном каталоге необ.файлы
  3. git init - создать новый репозиторий
  4. git add - добавить в очередь на коммит все файлы
  5. git commit -m 'Любой комментарий к комиту'
  6. Создать реп на Github
  7. Выбрать и скопировать ссылку SSH
  8. git remote add origin <скопированная_ссылка>
  9. git push -u origin master(main) (загрузить локальный реп на GitHub)


3. Как получить на локальный компьютер изменения с github.com

  Если не было предварительных коммитов - git clone <ssh ссылка на репо на github>
  Если были предварительные коммиты - git pull
  Если несколько разработчиков и возникает конфликт версий - git fetch, git merge

4. Какую информацию нам дают команда :
  git status - выводит состаяние рабочего каталога (если ничего не показывает, это значит что все изменения закомичены)
		(1) неотслеживаемые untracked
		(2) отслеживаемые   tracked
		(3) игнорируемые  ignored
  git log - показывает состояние коммитов в локальном репозитории
  git remote -v - показывает информацию о привязке локального репозитоя к репозиторию на github.com
